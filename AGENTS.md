# esp-pachislo-datacounter

ESP-IDF 6.1.0 project targeting **XIAO ESP32-C6** and **XIAO ESP32-C3** (`idf.py set-target esp32c6` or `esp32c3` before first build).

## Build / Flash / Monitor

```sh
idf.py set-target esp32c6           # (or esp32c3) one-time: generates sdkconfig
idf.py build                         # uses Ninja under build/
idf.py -p /dev/ttyACM0 flash monitor # serial flash + console
```

No test infrastructure exists in this repo. No CI, no pre-commit hooks.

## Configuration

All tunables in **`main/include/app_config.h`** — pin maps, debounce times, pulse coefficients, bonus labels, Wi-Fi creds, display refresh. Set unused GPIO pins to `-1`.

**Pin mapping:** `main/include/xiao_pinmap.h` translates XIAO `D0..D10` header labels to the correct GPIO number for each target (C3 vs C6). `app_config.h` uses `XIAO_Dx` macros so the same pin assignments work on both boards without modification.

**Wokwi simulation:** `WOKWI_DISABLE_WIRELESS` defaults to `1`. When set, both Wi-Fi and the web server are skipped at startup. Wokwi wiring is in `diagram.json` (Wokwi + Wokwi-compatible). To switch targets, change `"type": "board-xiao-esp32-c6"` to `"board-xiao-esp32-c3"`; the `esp:Dx` connections are resolved by Wokwi per board type.

## Architecture

Single `main/` component; entrypoint `app_main()` in `app_main.c`. Two FreeRTOS tasks:

| Task | Priority | Role |
|---|---|---|
| `signal_consumer` | 10 | Dequeues GPIO events → updates data model |
| `display_task` | 5 | Polls model + renders OLED every 500ms |

Source files:
- `input_monitor.c` — GPIO ISR + debounce → event queue
- `data_model.c` — atomic counter state (critical section guarded)
- `display_manager.c` — u8g2 OLED (SSD1306 128×64, I2C addr 0x3C)
- `wifi_manager.c` — STA mode with retry + 15 s timeout
- `web_server.c` — HTTP on port 80; routes `/` (dashboard HTML), `/api/stats` (JSON)

## GPIO signal quirks

| Signal | Interrupt type | Behavior |
|---|---|---|
| IN, OUT, Page button | `NEGEDGE` | Falling edge = active (open-drain). Counts on every active pulse. |
| BONUS1, BONUS2, DOOR, ERROR | `ANYEDGE` | Both edges tracked. `data_model_on_event(BONUS*)` fires **only on rising edge**. State synced before ISR enable (`input_monitor.c:100-106`). |

- Bonus interval game counter resets to 0 when **all** bonuses deactivate.
- Medal accumulation continues during bonus (never paused).

## OLED pages

Page switch button cycles 3 pages. Startup splash: ~3 s. Page title shown ~3 s after each switch. Uses `u8g2_font_unifont_t_japanese1`.

## LSP / clangd

Requires `build/compile_commands.json` (auto-generated by `idf.py build`). `.clangd` strips `-f*` and `-m*` flags. Espressif's custom clangd binary path is in `.vscode/settings.json`.

## Extension points

- **New display page** → add case in `display_manager.c:display_manager_render()`
- **Machine profile** → add `machine_profile` module, swap formulas in `data_model_on_event()`
- **Web API** → add handler + register in `web_server.c`
